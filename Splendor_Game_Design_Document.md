## 1. 游戏概述

### 1.1 游戏简介

Splendor（璀璨宝石）是一款由Marc André设计的策略桌游，于2014年发行。玩家扮演文艺复兴时期的宝石商人，通过收集宝石、购买发展卡、吸引贵族拜访来积累声望，最终成为最负盛名的宝石大亨。

### 1.2 核心设计理念

| 设计原则 | 描述 |
|----------|------|
| **简洁易懂** | 规则简单，每回合仅执行一个行动 |
| **策略深度** | 资源管理与引擎构建的完美结合 |
| **互动性** | 玩家间的资源竞争与卡牌争夺 |
| **节奏感** | 快节奏游戏，单局约30分钟 |

### 1.3 游戏规格

```
┌─────────────────────────────────────────────────────┐
│  游戏参数                                            │
├─────────────────────────────────────────────────────┤
│  玩家人数: 2-4人                                     │
│  游戏时长: 20-40分钟                                 │
│  适用年龄: 10岁以上                                  │
│  游戏类型: 资源管理 / 引擎构建 / 策略                │
│  胜利条件: 首先达到15分                              │
└─────────────────────────────────────────────────────┘
```

## 2. 游戏核心机制详解

### 2.1 宝石资源系统

#### 2.1.1 宝石类型与配置

游戏包含6种宝石代币，每种具有独特的颜色标识和战略价值：

```
┌────────────────────────────────────────────────────────────────┐
│                        宝石资源体系                              │
├──────────┬──────────┬──────────┬──────────┬──────────┬─────────┤
│  钻石     │  蓝宝石   │  祖母绿   │  红宝石   │  玛瑙     │  黄金    │
│ (White)  │  (Blue)  │ (Green)  │  (Red)   │ (Black)  │ (Gold)  │
├──────────┼──────────┼──────────┼──────────┼──────────┼─────────┤
│   ◇      │    ◆     │    ◈     │    ◇     │    ◆     │   ★     │
│  白色    │   蓝色    │   绿色    │   红色    │   黑色    │  金色   │
├──────────┴──────────┴──────────┴──────────┴──────────┴─────────┤
│  普通宝石 (可用于购买卡牌)                                        │
│  ────────────────────────────────────────────────────────────  │
│  黄金 (万能代币，仅用于支付差价，不可主动拿取)                      │
└────────────────────────────────────────────────────────────────┘
```

#### 2.1.2 宝石池配置规则

根据玩家人数动态调整宝石池初始数量：

| 玩家人数 | 每种普通宝石数量 | 黄金代币 | 贵族卡数量 |
|:--------:|:---------------:|:--------:|:---------:|
| 2人 | 4个 | 5个 | 3张 |
| 3人 | 5个 | 5个 | 4张 |
| 4人 | 7个 | 5个 | 5张 |

**配置逻辑说明**：
```
宝石数量 = 基础值 + (玩家人数 - 2) × 增量
普通宝石基础值: 4, 增量: 1.5 (取整)
黄金代币: 固定5个 (用于预订卡牌)
```

#### 2.1.3 宝石持有上限

```
┌─────────────────────────────────────────┐
│           宝石持有规则                    │
├─────────────────────────────────────────┤
│  普通宝石上限: 10个                       │
│  黄金代币: 不计入上限                      │
│  超出上限时: 必须归还宝石至宝石池           │
│  归还规则: 玩家自主选择归还哪些宝石         │
└─────────────────────────────────────────┘
```

### 2.2 发展卡系统

#### 2.2.1 卡牌结构

每张发展卡包含以下属性：

```
┌─────────────────────────────────────┐
│  ┌─────────────────────────────┐    │
│  │         卡牌ID              │    │
│  │      [Tier-Level]           │    │
│  │                             │    │
│  │    ┌─────────────────┐      │    │
│  │    │    卡牌图案      │      │    │
│  │    │   (视觉主题)     │      │    │
│  │    └─────────────────┘      │    │
│  │                             │    │
│  │  ┌───┐                      │    │
│  │  │ ◇ │ ×2  成本要求         │    │
│  │  │ ◆ │ ×1                   │    │
│  │  └───┘                      │    │
│  │                             │    │
│  │  ┌───┐      ┌───┐          │    │
│  │  │ ◈ │      │ 2 │ 分值     │    │
│  │  └───┘      └───┘          │    │
│  │  折扣颜色                   │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

**属性详解**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | String | 唯一标识符，格式为 `t{tier}-{index}` |
| `tier` | Integer | 卡牌等级 (1/2/3) |
| `cost` | Object | 购买所需的宝石数量 |
| `points` | Integer | 胜利点数 (0-5) |
| `color` | Enum | 提供的永久折扣颜色 |

#### 2.2.2 折扣机制

发展卡的核心价值在于提供永久折扣：

```
折扣计算公式:
───────────────────────────────────────────────────────
实际支付 = max(0, 卡牌成本 - 玩家持有该颜色卡牌数)

示例:
  卡牌成本: { white: 3, blue: 2, green: 1 }
  玩家持有: 2张白色卡 + 1张蓝色卡
  实际支付: { white: 1, blue: 1, green: 1 }
───────────────────────────────────────────────────────
```

**折扣效果示意**：

```
购买前:                    购买后:
┌─────────────┐           ┌─────────────┐
│ 玩家持有:    │           │ 玩家持有:    │
│ ◇◇◇ ◆◆ ◈    │           │ ◇◇◇ ◆◆ ◈    │
│             │           │             │
│ 发展卡:      │   ───►    │ 发展卡:      │
│ 红色折扣 ×1  │           │ 红色折扣 ×2  │
│ (未来红宝石  │           │ (未来红宝石  │
│  成本-1)     │           │  成本-2)     │
└─────────────┘           └─────────────┘
```

### 2.3 贵族令牌系统

#### 2.3.1 贵族卡结构

```
┌─────────────────────────────────────┐
│         贵族卡结构                    │
├─────────────────────────────────────┤
│  ┌─────────────────────────────┐    │
│  │      贵族肖像图案            │    │
│  │                             │    │
│  │  ┌───┬───┬───┬───┬───┐      │    │
│  │  │ ◇ │ ◆ │ ◈ │ ◇ │ ◆ │      │    │
│  │  │ 3 │ 0 │ 3 │ 0 │ 3 │ 需求  │    │
│  │  └───┴───┴───┴───┴───┘      │    │
│  │                             │    │
│  │         ┌───┐               │    │
│  │         │ 3 │ 分值          │    │
│  │         └───┘               │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

#### 2.3.2 贵族拜访规则

```
触发条件:
───────────────────────────────────────────────────────
玩家购买发展卡后，检查是否满足任意贵族卡的要求

判定逻辑:
  对于每张贵族卡:
    如果 玩家每种颜色发展卡数量 >= 贵族卡需求数量:
      触发贵族拜访

拜访效果:
  - 玩家立即获得3分
  - 贵族卡从公共区域移至玩家区域
  - 每位玩家最多获得多张贵族卡
───────────────────────────────────────────────────────
```

**贵族卡需求分布**：

| 需求类型 | 示例 | 数量分布 |
|----------|------|----------|
| 三色各3张 | {white:3, green:3, black:3} | 4张 |
| 四色各2张 | {white:2, blue:2, green:2, red:2} | 3张 |
| 单色4张+其他 | {black:4, red:3} | 3张 |

### 2.4 预留卡牌机制

#### 2.4.1 预留规则

```
┌─────────────────────────────────────────┐
│           预留卡牌规则                    │
├─────────────────────────────────────────┤
│  预留上限: 每位玩家最多3张                │
│  预留来源:                               │
│    - 牌堆顶部的暗牌                      │
│    - 公开展示的明牌                      │
│  预留奖励: 获得1个黄金代币 (如果还有)      │
│  购买限制: 仅可购买自己预留的卡或公开展示的卡│
└─────────────────────────────────────────┘
```

#### 2.4.2 预留策略价值

```
预留卡牌的战略意义:
───────────────────────────────────────────────────────
1. 锁定高价值卡牌
   - 防止对手抢先购买
   - 为后续购买做准备

2. 获得黄金代币
   - 黄金可作为任意颜色宝石使用
   - 提供支付灵活性

3. 信息隐藏
   - 暗牌预留增加不确定性
   - 干扰对手策略判断
───────────────────────────────────────────────────────
```

#### 5.2.2 AI决策框架

```javascript
// AI 决策核心框架
class SplendorAI {
  constructor(difficulty = 'medium') {
    this.difficulty = difficulty;
    this.weights = this.initializeWeights();
  }

  // 行动评估函数
  evaluateAction(gameState, action, playerIndex) {
    let score = 0;
    
    switch (action.type) {
      case 'take_gems':
        score = this.evaluateTakeGems(gameState, action, playerIndex);
        break;
      case 'buy_card':
        score = this.evaluateBuyCard(gameState, action, playerIndex);
        break;
      case 'reserve_card':
        score = this.evaluateReserveCard(gameState, action, playerIndex);
        break;
    }
    
    return score;
  }

  // 评估拿取宝石行动
  evaluateTakeGems(gameState, action, playerIndex) {
    const player = gameState.players[playerIndex];
    let score = 0;
    
    // 评估宝石对后续购买的价值
    for (const gem of action.gems) {
      score += this.calculateGemValue(gameState, player, gem.color);
    }
    
    // 考虑宝石上限
    const totalGems = Object.values(player.gems).reduce((a, b) => a + b, 0);
    if (totalGems + action.gems.length > 10) {
      score -= this.weights.gemLimitPenalty;
    }
    
    return score;
  }

  // 评估购买卡牌行动
  evaluateBuyCard(gameState, action, playerIndex) {
    const player = gameState.players[playerIndex];
    const card = action.card;
    let score = 0;
    
    // 分值收益
    score += card.points * this.weights.pointValue;
    
    // 折扣收益 (长期价值)
    score += this.weights.bonusValue;
    
    // 贵族卡匹配度
    score += this.calculateNobleProgress(gameState, player, card);
    
    // 成本效率
    const efficiency = card.points / this.calculateActualCost(player, card);
    score += efficiency * this.weights.efficienyBonus;
    
    return score;
  }

  // 评估预留卡牌行动
  evaluateReserveCard(gameState, action, playerIndex) {
    const player = gameState.players[playerIndex];
    const card = action.card;
    let score = 0;
    
    // 高价值卡牌预留优先级
    if (card.points >= 3) {
      score += this.weights.highValueReserve;
    }
    
    // 阻止对手获得关键卡
    score += this.calculateDenialValue(gameState, card);
    
    // 黄金代币价值
    if (gameState.gemPool.gold > 0) {
      score += this.weights.goldValue;
    }
    
    // 预留上限惩罚
    if (player.reservedCards.length >= 2) {
      score -= this.weights.reserveLimitPenalty;
    }
    
    return score;
  }

  // 选择最佳行动
  selectBestAction(gameState, playerIndex) {
    const actions = this.getLegalActions(gameState, playerIndex);
    let bestAction = null;
    let bestScore = -Infinity;
    
    for (const action of actions) {
      const score = this.evaluateAction(gameState, action, playerIndex);
      if (score > bestScore) {
        bestScore = score;
        bestAction = action;
      }
    }
    
    return bestAction;
  }
}
```

#### 5.2.4 高级AI策略 - 蒙特卡洛树搜索

```
MCTS 算法流程:
──────────────────────────────────────────────────────────────────

     ┌─────────────────────────────────────┐
     │            选择 (Selection)          │
     │  从根节点开始, 选择UCB值最高的子节点 │
     │  直到到达未完全展开的节点            │
     └──────────────────┬──────────────────┘
                        │
                        ▼
     ┌─────────────────────────────────────┐
     │            扩展 (Expansion)          │
     │  为选中的节点添加一个或多个子节点    │
     │  每个子节点代表一个可能的行动        │
     └──────────────────┬──────────────────┘
                        │
                        ▼
     ┌─────────────────────────────────────┐
     │          模拟 (Simulation)           │
     │  从新节点开始, 随机进行游戏模拟      │
     │  直到达到终止状态                    │
     └──────────────────┬──────────────────┘
                        │
                        ▼
     ┌─────────────────────────────────────┐
     │          回溯 (Backpropagation)      │
     │  将模拟结果反向传播到所有祖先节点    │
     │  更新节点的访问次数和胜率            │
     └──────────────────┬──────────────────┘
                        │
                        ▼
     ┌─────────────────────────────────────┐
     │          重复迭代                    │
     │  直到达到时间或迭代次数限制          │
     │  选择访问次数最多的子节点作为最佳行动│
     └─────────────────────────────────────┘

UCB公式:
UCB1 = W/N + c * sqrt(ln(P)/N)

其中:
- W: 节点获胜次数
- N: 节点访问次数
- P: 父节点访问次数
- c: 探索常数 (通常为sqrt(2))
──────────────────────────────────────────────────────────────────
```
